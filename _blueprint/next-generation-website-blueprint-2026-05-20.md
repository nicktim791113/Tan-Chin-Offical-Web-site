# 震欣科技新版網站完整設計藍圖

日期：2026-05-20
目標：建立一個不參考舊站視覺、屬於震欣科技自己的新世代官方網站。
方向：未來感、AI 機器人感、金屬感、精密製造感、國際級工業品牌感。

## 一、核心設計概念

新版網站的主軸：

```txt
Metal Intelligence
由 AI 與精密製程驅動的金屬射出成型能力
```

它要讓訪客在 10 秒內感覺到：

- 震欣不是傳統工廠網站
- 震欣懂精密金屬零件
- 震欣有能力處理複雜幾何、特殊材料、客製開發
- 震欣值得被國際買主、研發工程師、採購窗口信任
- 客戶會想立刻送圖詢問可製性

## 二、視覺風格

### 風格關鍵字

```txt
AI
Robot
Metal
MIM
Precision
Inspection

```

中文描述：

```txt

不是傳統型錄網站，而是一個高階工業技術展示平台。
```

### 要避免的感覺

- 不要看起來像模板網站
- 不要一進來就是很長的公司簡介

### 要保留的精神

- 可信任
- 工業專業
- 精密
- 客製能力
- 台灣製造實力
- 金屬材質真實感

## 五、首頁設計藍圖

### 第一屏：AI 金屬成型主視覺

畫面概念：

```txt
深色金屬空間中，精密金屬零件被 AI 掃描線分析。
零件像是從粉末、數據、機器視覺中組裝出來。
背景有非常細緻的格線、掃描光、金屬反射。
```

主標題建議：

```txt
AI-Ready Metal Injection Molding
為未來機械與精密零件打造的金屬射出成型
```

副標：

```txt
震欣科技專注於 MIM 金屬射出成型，協助客戶將複雜形狀、特殊材料與高精度需求轉化為可量產的金屬零件。
```

CTA：

```txt
開始詢問製造可行性
探索材料能力
```

### 第二屏：能力數據

用精密儀表板方式呈現：

```txt
MIM 複雜形狀成型
重量可達 100g
1000+ 客製化零件開發經驗
ISO 9001:2015
射出 / 脫脂 / 燒結 / 加工 / 檢驗
X-RAY / 品質檢測能力
```

視覺：

- 金屬卡片
- 極細線框
- 輕微掃描動畫
- 數據像工程儀表

### 第三屏：MIM 製程流程

做成橫向或垂直互動流程：

```txt
1. 金屬粉末與結合劑
2. 混鍊與造粒
3. 射出成型
4. 脫脂
5. 真空燒結
6. 再加工與表面處理
7. 檢驗與品質確認
```

每一階段顯示：

- 一句說明
- 對應圖片或 3D icon
- 此階段控制的品質風險

### 第四屏：材料矩陣

材料分類：

```txt
鎳合金鋼
鉻鉬合金鋼
鎳鉻鉬合金鋼
不鏽鋼 17-4PH
不鏽鋼 316L
高速鋼
其他材質
```

呈現方式：

- 像一個 AI 材料選擇介面
- 滑過卡片時顯示材料特性
- 點擊進入對應材料頁
- 每個材料頁展示產品照片與應用方向

### 第五屏：產品與樣品展示

重點：

- 不只是圖片格子
- 要像精密零件 gallery
- 可以依材料篩選
- 每張圖可以放大
- 每個分類都有「詢問此類零件」按鈕

### 第六屏：品質與檢測

內容方向：

```txt
品質不是最後檢查，而是每個製程階段的控制系統。
```

可展示：

- ISO 證書
- 量測設備
- 磁粉探傷
- X-RAY / CT 設備
- 製程紀錄
- 抽檢與巡檢概念

### 第七屏：聯絡與詢價

不要只是傳統表單，要像專案啟動：

```txt

姓名
```

## 八、技術架構建議

首選：

```txt
Astro + TypeScript + Markdown/JSON + 少量 Three.js
```

原因：

- 震欣網站以內容、產品、公司能力展示為主
- 靜態網站速度快、安全、維護成本低
- Astro 預設就適合做靜態內容網站
- 需要互動的地方才載入 JavaScript
- 首頁可以局部使用 Three.js 做震撼視覺
- 不會像整站 React 那樣過重

備選：

```txt
Next.js App Router + Static Export
```

如果未來要做比較完整的互動應用、後台、客戶專區，可以考慮 Next.js。

## 九、3D / 動畫策略

可以做，但要克制。

建議使用：

- 首頁第一屏 3D 金屬零件場景
- 材料卡片微互動
- 製程流程掃描動畫
- 產品圖片 hover 放大
- 品質檢測區掃描線

不要做：

- 讓 3D 影響讀取速度
- 手機版硬跑大型 3D
- 整頁炫技但內容難讀
- 炫光過多導致廉價感

技術原則：

```txt
桌機顯示完整 3D
手機使用輕量動畫或靜態替代圖
使用者開啟 reduced motion 時關閉大型動畫
```

### 第四階段：互動與 3D

- 加入 Three.js hero

### 第五階段：表單與部署

- 建立聯絡表單

- 部署測試站
- 加入 sitemap / SEO / redirect

### 第六階段：DNS 切換

- 新站先在測試網址確認
- 保留舊站作備援
- 確認表單可收信
- 確認 email DNS 不受影響
- 從 HiNet 更新 `www` 指向
- 觀察 24 至 48 小時
