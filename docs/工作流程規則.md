# 震欣科技網站工作流程規則

本文件是之後每次修改、推播、拉回時的共同規則。

## 開工前檢查

1. 確認目前資料夾為 `Tan-Chin-Offical-Web-site`。
2. 執行 `git status --short --branch`。
3. 確認 `origin` 指向 `Tan-Chin-Offical-Web-site` GitHub 儲存庫。
4. 執行 `git pull --ff-only`。
5. 若 pull 失敗、發生衝突、或本機有未提交變更，先停止並回報，不強制覆蓋。

## 修改後流程

1. 執行備份腳本：

   ```powershell
   powershell -NoProfile -ExecutionPolicy Bypass -File ".\scripts\backup-core-assets.ps1"
   ```

2. 依修改幅度提升版本號：
   - patch：小修正、文字調整、bug 修正。
   - minor：新功能或較大改善。
   - major：大改版或不相容變更。

3. 同步更新版本：
   - `package.json`
   - `package-lock.json`
   - `future-site/package.json`
   - `future-site/package-lock.json`
   - 若程式中有顯示版本號，也一併更新。

4. 更新正式更新紀錄：
   - `docs/更新紀錄.md`
   - 新增今天日期、版本號、修改摘要。

5. 檢查：
   - `git diff`
   - `git status --short --branch`
   - 必要測試，例如 `npm run check`、`npm run build`。

6. 不上傳：
   - `node_modules/`
   - `dist/`
   - `_backups/`
   - `_verification/`
   - SQLite 實際資料庫檔
   - log 檔
   - `.env` 或密碼金鑰檔案

7. 檢查正常才 commit 與 push：

   ```powershell
   git add -A
   git commit -m "v版本號：本次修改摘要"
   git push
   ```

## 部署策略

目前先使用 GitHub Pages 建置靜態官網。未來若要新增 AI 詢價、圖面上傳、客戶入口或伺服器端 API，可再串接 Vercel 或獨立後端服務。

## Vercel 自動部署規則

1. Vercel 已連接 GitHub 儲存庫 `nicktim791113/Tan-Chin-Offical-Web-site`。
2. Vercel 專案根目錄為 `future-site`。
3. 推播到 `main` 後，Vercel 會自動執行 Production Deployment。
4. 其他 branch 或 Pull Request 可作為 Preview Deployment。
5. Vercel 部署設定記錄於 `future-site/vercel.json`，相關觀念與流程記錄於 `docs/Vercel部署流程.md`。
6. 若未來要接正式網域，優先把 `www.tanchin.com.tw` 指向 Vercel；GitHub Pages 保留為備援或展示用途。
